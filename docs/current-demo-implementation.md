# 当前 Demo 实现说明

## 1. 当前版本定位

当前 Demo 用于验证“权签票据一致性 AI 预审”的完整轻量闭环：

```text
付款任务
→ 系统支付指令
→ 票据文件
→ 模板规则
→ 多模态模型提取票面字段
→ 字段归一
→ 规则核验
→ 风险展示
→ 人工反馈
→ 模板调优
```

Demo 不使用公司真实票据或真实 MAP 数据，样例全部为合成数据。模型接口按 OpenAI-compatible 协议接入，当前已用火山方舟 `Doubao-Seed-2.0-pro` 完成端到端验证。

## 2. 页面结构

当前前端采用四个标签页组织用户旅程。

### 2.1 付款核验

这是主展示页，面向权签人。

用户流程：

1. 选择一笔样例付款任务。
2. 页面展示系统支付指令。
3. 页面展示票面文档。
4. 页面展示当前模板。
5. 点击“开始 AI 预审”。
6. 后端调用多模态模型识别票面字段。
7. 后端根据字段 Schema 和模板调优配置归一字段。
8. 后端执行一致性比对。
9. 页面默认只展示风险项，通过项可展开查看。
10. 权签人可提交确认、忽略、AI 识别错误或优化反馈。

### 2.2 模板调优

面向运营、AI 产品或财务配置人员。

这里按模板维护 AI 提取时需要参考的业务知识：

- 票面常见叫法。
- 位置/模板提示。
- AI 识别要求。
- 字段比对口径。

这些内容会进入模型提取 prompt，帮助模型把票面字段映射到标准字段。

### 2.3 反馈样本

用于承接权签人的人工反馈。

当前版本已经支持保存反馈到本地 `data/feedback/`，页面先保留反馈样本池入口。后续可以增加反馈列表接口，聚合高频错误并转成模板调优建议。

### 2.4 系统设置

用于一次性配置模型连接信息：

- OpenAI-compatible URL。
- 模型名称。
- API Key。
- 超时时间。
- 文本测试。
- 图片测试。

配置保存到本地 `config.local.json`，该文件被 `.gitignore` 忽略，不会提交到 Git。

## 3. 核心实现

### 3.1 后端

后端使用 FastAPI，主要模块：

- `main.py`：API 路由和静态页面托管。
- `model_client.py`：OpenAI-compatible 模型调用封装。
- `extractor.py`：构造 prompt、调用模型、解析模型结构化结果。
- `comparator.py`：金额、账号、币种、日期、文本等比对规则。
- `schemas.py`：请求、响应和提取结果 Schema。
- `storage.py`：配置、样例、反馈和本地模型设置读写。

### 3.2 前端

当前前端是轻量原生 HTML/CSS/JavaScript，由 FastAPI 直接托管。

这样做是为了快速验证产品旅程，减少 React/Vite 工程复杂度。后续如果要演进为正式前端，可以把现有四个标签页迁移为 React 路由或组件。

### 3.3 配置文件

- `config/field_schema.json`：标准字段、风险等级、比对方式。
- `config/field_aliases.json`：字段常见叫法。
- `config/mapping_rules.json`：模板规则和位置提示。
- `config.local.json`：本地模型连接配置，不提交。

## 4. 样例数据

当前包含三组样例：

1. 国内支票整体一致。
2. 跨境电汇金额不一致。
3. 跨境电汇收款账号不一致。

样例包括：

- 合成票据 PNG。
- 模拟 MAP/AP 系统支付指令 JSON。
- 预置期望提取结果 JSON。

当前主流程会调用真实模型进行票面字段提取；预置结果主要用于对照和兜底。

## 6. 当前限制

- 反馈样本池页面还没有聚合展示历史反馈。
- 模板识别当前主要依赖样例任务中的 `template_id`，还没有实现“系统自动匹配 + AI 辅助识别 + 人工修正”的完整流程。
- 模板调优规则当前通过 prompt 影响模型提取，后端规则兜底映射还可以继续增强。
- 文件上传当前主要支持图片预览和模型识别；PDF 转图片能力尚未接入主流程。
- 目前没有用户权限、任务列表、审计记录和 MAP 真实接口。

## 7. 下一步建议

建议下一轮优先做：

1. 模板识别流程：系统带出模板、AI 建议模板、人工修正模板。
2. 反馈样本池：展示反馈、聚合高频错误、转成模板调优建议。
3. AI 调优闭环：反馈样本一键生成字段别名、位置提示或识别要求。
4. PDF 上传预处理：PDF 转 PNG 后进入模型识别。
5. 更接近 MAP 的系统指令录入和字段分组展示。
