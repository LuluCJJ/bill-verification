# 网页与脚本调用差异排查

如果 `scripts/test_model_api.py` 的文本测试可以跑通，但网页 Demo 跑不通，通常不是“模型一定不可用”，而是两条调用链路不同。

## 两条链路的差异

脚本链路更短：

```text
test_model_api.py -> OpenAI-compatible 模型接口
```

网页链路多了本地应用层：

```text
浏览器页面 -> 本地 FastAPI 后端 -> 本地模型配置 -> OpenAI-compatible 模型接口 -> JSON 解析 -> 规则核验 -> 前端展示
```

所以脚本文本测试成功，只能证明“当前机器可以用某种配置发起一次文本模型请求”。网页端还会额外受到本地服务、配置来源、浏览器访问、图片格式、模型返回 JSON 质量等因素影响。

## 常见原因

1. **配置来源不同**

   网页端读取 `config.local.json` 或环境变量中的配置；测试脚本通常通过命令行参数、环境变量或 `.env` 读取配置。两边可能不是同一份 URL、模型名、超时时间或 API Key。

2. **网页端必须先启动本地后端**

   页面不是直接调用模型，而是调用 `http://127.0.0.1:8000/api/...`。如果公司电脑限制本地端口、浏览器无法访问本地服务，或者后端没有启动，网页就会失败。

3. **终端和浏览器/后端的网络环境可能不同**

   有些公司电脑会给命令行、浏览器、Python 进程配置不同代理策略。脚本能访问模型，不代表由本地 FastAPI 进程发起的请求也一定走同样的代理。

4. **文本测试和图片测试不是同一类能力**

   文本测试只验证 chat completion 的纯文本请求。图片测试还涉及图片读取、base64 编码、MIME 类型、接口对 `image_url` 的兼容方式，以及模型是否支持多模态输入。

5. **图片格式要求更严格**

   火山方舟接口此前对 `image_url` 报过 `Invalid base64 image_url`。当前 Demo 应使用 PNG/JPG/WebP 这类常见位图格式，不建议直接把 SVG/PDF 当图片传给模型。

6. **网页端需要模型返回可解析 JSON**

   主流程不是只问模型一句话，而是要求模型按照字段结构返回 JSON。即使文本问答成功，如果模型返回了额外说明、Markdown 包裹或不完整 JSON，后续结构化解析也可能失败。

7. **Python 环境可能不是同一个**

   项目建议使用 `.venv`。如果脚本是在系统 Python 下跑通，而网页服务是在 `.venv` 下启动，两边依赖版本、代理环境和证书环境都可能不同。

## 建议排查顺序

1. 启动 Demo：

   ```bat
   start_demo.bat
   ```

2. 打开健康检查：

   ```text
   http://127.0.0.1:8000/api/health
   ```

   能返回正常结果，才说明本地后端已启动。

3. 打开模型配置检查：

   ```text
   http://127.0.0.1:8000/api/model/settings
   ```

   确认 `base_url` 和 `model` 已保存。页面不会回显 API Key，这是正常的。

4. 在 **系统设置** 页先点“测试文本”。如果失败，优先检查 URL、模型名、API Key、代理和后端日志。

5. 点“诊断链路”。诊断会按步骤显示本地后端、模型配置、文本模型调用、图片模型调用是否通过，并展示调用 endpoint、模型名、Key 是否已保存、重试次数和模型返回摘要。诊断不会回显 API Key。

6. 再点“测试图片”。如果文本成功但图片失败，优先检查模型是否支持图片、图片格式是否为 PNG/JPG/WebP，以及后端日志中的模型错误。

7. 最后到 **付款核验** 页点击“开始 AI 预审”。如果图片测试成功但主流程失败，重点看模型返回内容是否为可解析 JSON。

## 如何解读诊断结果

- **本地后端失败**：页面没有连上本机 FastAPI，通常是服务没启动、端口被占用，或公司电脑限制本地端口访问。
- **模型配置失败**：`config.local.json` 没有保存完整配置，通常是 API Key 没填、保存失败，或页面和脚本读取的不是同一套配置。
- **文本模型调用失败**：本地后端已经启动，但 Python 后端访问模型接口失败。常见原因是模型名/URL/权限错误、公司代理差异、证书或网络阻断。
- **图片模型调用失败**：文本链路可用，但多模态图片链路失败。常见原因是模型不支持图片、图片格式不被支持、base64 image_url 协议不兼容，或图片过大/格式异常。
- **偶发 ConnectError**：如果重试后成功，说明配置本身大概率没问题，更像公司网络、代理或模型网关的瞬时连接问题。

## 脚本侧图片测试

可以用合成样例先单独测图片能力：

```bat
.venv\Scripts\python.exe scripts\test_model_api.py --image data\samples\documents\model_test_tt.png --prompt "请识别这张付款文件中的金额、币种、收款方和账号，返回简洁 JSON。"
```

如果脚本图片测试也失败，说明问题更靠近模型接口、图片格式或多模态协议；如果脚本图片测试成功但网页图片测试失败，再回到本地服务配置、代理和后端日志排查。
